# NeighborlyWork 3.3 Gap Audit — Contractor Pricing Profiles + Draft Quote Generation

_Status: opened in parallel on 2026-04-24 after 3.2 implementation shipped but before 3.2 verification fully closed._

## What is already present
- Schema proposal includes `public.contractor_pricing_profiles` for rule-based quote draft generation inputs.
- `supabase/001_backend_v1_schema.sql` already creates `public.contractor_pricing_profiles`.
- `public.quotes` already has fields that could support draft/generated quote structure:
  - `status`
  - `system_type`
  - `system_size_tons`
  - `install_timeline_days`
  - `price_total`
  - `price_breakdown`
  - `draft_generated_at`

## What is missing in the app today
- No app page currently reads or writes `contractor_pricing_profiles`.
- No app flow currently generates quote drafts from pricing rules.
- Contractor portal still relies on manual quote entry only:
  - equipment brand/model
  - seer
  - warranty
  - labor/equipment/additional cost
  - manual total calculation
- No founder/admin workflow exists yet for reviewing pricing-profile completeness.

## Immediate implications
- Checkpoint 3.3 is not blocked by schema absence.
- It is blocked by missing service/app logic.
- The clean next implementation surface is likely:
  1. contractor pricing profile CRUD path
  2. profile completeness gate on contractor accounts
  3. server/service logic to create draft quote suggestions from profile + brand ratings
  4. UI path in contractor portal to accept/edit/promote a generated draft into a submitted quote

## Recommended next build slice
1. Add contractor pricing profile read/write path.
2. Add contractor portal screen for profile setup.
3. Persist `pricing_profile_complete` on `public.contractors`.
4. Add first-pass deterministic quote draft function using:
   - base labor rate
   - brand markups
   - service area fees
   - default warranty
   - default install timeline
5. Keep final contractor review/edit step before any draft becomes `submitted`.
